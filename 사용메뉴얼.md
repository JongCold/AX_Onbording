## 5. 자동화 프로세스 사용 매뉴얼 및 서비스 동작 순서

최신 업데이트된 아키텍처를 바탕으로 전체 서비스의 구동 순서 및 사용 방법을 안내합니다.

### 5.1. 신입사원 온보딩 시작 (POST `/onboard`)
대표/관리자가 신규 근로자 정보(이름, 이메일, 프로젝트명)를 등록하면 백그라운드 프로세스가 가동되어 슬랙 채널을 자동 생성하고 인프라를 세팅합니다.

* **API Endpoint**: `POST http://localhost:8000/onboard`
* **Content-Type**: `application/json; charset=utf-8`
* **요청 바디 (Request Body)**:
  ```json
  {
      "name": "홍길동",
      "email": "gildong@example.com",
      "project_name": "강북창업지원센터"
  }
  ```
* **호출 예시 (한글 깨짐 방지)**:
  PowerShell에서 기본 인코딩 문제로 한글(`???`)이 깨지는 것을 방지하기 위해 아래와 같이 바이트 배열 방식이나 `curl.exe`를 사용합니다.
  ```powershell
  # 방법 1: PowerShell UTF-8 강제 적용
  $body = '{"name": "홍길동", "email": "gildong@example.com", "project_name": "강북창업지원센터"}'
  $bodyBytes = [System.Text.Encoding]::UTF8.GetBytes($body)
  Invoke-RestMethod -Uri "http://localhost:8000/onboard" -Method Post -Body $bodyBytes -ContentType "application/json; charset=utf-8"

  # 방법 2: curl.exe 사용
  curl.exe -X POST http://localhost:8000/onboard -H "Content-Type: application/json; charset=utf-8" -d "{\`"name\`": \`"홍길동\`", \`"email\`": \`"gildong@example.com\`", \`"project_name\`": \`"강북창업지원센터\`"}"
  ```

### 5.2. 백그라운드 채널 세팅 프로세스 (1단계 동작)
API 호출 즉시 아래의 작업이 비동기 백그라운드로 처리됩니다.
1. **채널 자동 생성**: 입력된 프로젝트명과 근로자 이름을 조합하여 `ot-강북창업지원-홍길동` 형식의 공개 채널을 자동 생성합니다. (이미 존재하면 해당 ID를 가져옵니다.)
2. **슬랙 초대 및 매핑**: 해당 이메일로 슬랙 유저를 검색하여 즉시 채널에 초대합니다. 가입 전이거나 조회가 불가능할 경우, 로컬 데이터베이스(`pending_workers.json`)에 대기자로 등록해 둡니다.
3. **가이드 문서 자동 배포**: 로컬에 있는 `2. 과업지시서.pdf`와 `온보딩.pdf` (없을 시 `온보딩_안내.txt` 자동 생성)를 채널에 파일 형태로 직접 업로드합니다.
4. **초기 웰컴 메시지 발송**: 채널 내에 과업 내용 확인을 당부하며, 챗봇 사용법(`@Auto_Bot1`)을 알리는 환영 메시지가 전송됩니다.

### 5.3. 신입사원 슬랙 가입 감지 (2단계 동작 - `team_join`)
1. 슬랙 워크스페이스에 아직 가입하지 않았던 사원이 나중에 초대 링크를 통해 가입을 완료하면, 슬랙 API가 `team_join` 이벤트를 서버로 쏩니다.
2. 서버는 `pending_workers.json` 대기자 명단을 조회하여, 해당 사원의 이메일이 매칭되면 배정된 채널로 즉시 자동 초대합니다.
3. 추가적으로 채널에 합류한 인원을 위해 맞춤형 웰컴 안내문을 재생성하여 게시합니다.

### 5.4. 슬랙 챗봇 활용 질의응답 (RAG 챗봇 동작 순서)
신입사원이 채널 내에서 챗봇(`@Auto_Bot1`)을 멘션하여 질문을 던지면 다음 순서로 답변이 생성됩니다.

1. **질문 작성**: 사원이 슬랙 채널에서 챗봇을 호출합니다.
   * 예시: `@Auto_Bot1 이번 과업의 산출물이 무엇인가요?`
2. **이벤트 수신 및 중복 차단 (`main.py`)**: 
   * 슬랙 서버에서 `app_mention` (또는 DM 시 `message`) 이벤트를 수신합니다.
   * **(핵심 방어 로직)** 슬랙의 타임아웃 재시도(Retry) 신호를 무시하고, `client_msg_id`를 인메모리 캐시(`PROCESSED_MSG_IDS`)에 대조하여 중복된 메시지 처리를 즉각 차단합니다.
   * 챗봇 처리를 백그라운드(`async_rag_chat_process`)로 넘기고 슬랙 서버에는 즉시 `200 OK`를 응답하여 루프를 방지합니다.
3. **RAG 컨텍스트 검색 (`rag_service.py`)**: 
   * 질문을 벡터(Vector)로 임베딩한 뒤, 사전에 파싱된 `2. 과업지시서.pdf` 기반의 로컬 JSON 벡터 DB에서 가장 유사도가 높은(0.15 이상) 문맥을 추출합니다.
4. **로컬 LLM 답변 생성 및 발송**: 
   * 추출된 문맥과 사원의 질문을 로컬 Ollama 모델(`gemma`)에 주입하여 환각 현상(Hallucination)이 없는 정교하고 정중한 한국어 답변을 생성합니다.
   * 생성된 답변은 질문이 올라온 슬랙 메시지의 **스레드(Thread)** 답글 형태로 발송됩니다.

# AX 온보딩 자동화 및 RAG 챗봇 시스템 실행 매뉴얼

본 매뉴얼은 단기 근로자 온보딩 대시보드 웹 애플리케이션 및 슬랙 RAG 챗봇 시스템을 로컬 환경에서 올바르게 설정하고 실행하기 위한 가이드입니다.

---

## 1. 사전 준비 사항

### 1.1 Python 환경 구성 및 패키지 설치
Python 3.10 이상의 환경이 권장됩니다.
프로젝트 루트 폴더(`c:\auto_onbording`)에서 아래 명령어를 실행하여 필수 의존성 패키지를 설치합니다.

```bash
pip install -r requirements.txt
```

### 1.2 .env 환경 변수 설정
루트 폴더에 위치한 `.env` 파일에 다음 항목들이 제대로 입력되어 있는지 확인합니다.

```env
# Slack API Credentials
SLACK_BOT_TOKEN=xoxb-xxxx...
SLACK_USER_TOKEN=xoxp-xxxx...
SLACK_SIGNING_SECRET=xxxx...

# SMTP Email (Gmail 등)
SMTP_USER=example@gmail.com
SMTP_PASSWORD=앱비밀번호(16자리)

# Google Drive OAuth Credentials
GOOGLE_CLIENT_ID=xxxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOPSP-xxxx...
GOOGLE_DRIVE_FOLDER_ID=구글드라이브_폴더_ID

# Gemini API (웰컴 환영 인사말 생성용)
GEMINI_API_KEY=AIzaSy...

# Ollama 설정 (로컬 RAG 검색용)
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=gemma2:2b
OLLAMA_EMBED_MODEL=nomic-embed-text
```

## 2. 외부 서비스 사전 실행 및 인증 설정

### 2.1 로컬 Ollama 엔진 실행
본 시스템은 로컬 임베딩 및 검색을 위해 Ollama를 사용합니다.
1. Ollama가 백그라운드에 켜져 있는지 확인합니다.
2. 터미널에서 다음 명령어를 통해 사용되는 모델들을 사전에 풀(Pull) 받아야 합니다.
   ```bash
   ollama pull nomic-embed-text
   ollama pull gemma2:2b
   ```

### 2.2 구글 드라이브 API 인증서 재갱신 (권한 추가)
신규 업로드 기능 개발로 인해 드라이브 권한(Scopes)이 변경되었습니다.
1. 기존에 발급받은 인증 토큰 파일 `token.json`이 있다면 **반드시 삭제**해 주십시오. (기존 토큰은 `readonly` 전용이라 업로드 시 권한 에러가 발생합니다)
2. 서버 구동 후 최초 1회 브라우저 팝업이 뜨며, 구글 계정으로 로그인 후 **드라이브 업로드 및 관리 권한** 동의를 승인해 주시면 `token.json`이 새롭게 자동 발급됩니다.

터미널에서 아래 명령어를 입력하여 FastAPI 서버를 구동합니다. Windows 환경의 멀티프로세싱 호환성 및 다국어 인코딩 문제를 해결하기 위해 `--reload` 옵션을 제외하고 단일 구동하는 방식을 권장합니다.

## 4. 온보딩 대시보드 웹 사용 가이드

서버가 켜지면 관리자(대표님)는 브라우저를 통해 손쉽게 신규 근로자를 등록할 수 있습니다.

1. **대시보드 접속**: 브라우저에서 아래 주소로 접속합니다.
   * `http://localhost:8000/`
2. **근로자 정보 기입**: 
   * **근로자 이름**, **이메일 주소**, **과업 프로젝트명**을 입력합니다.
3. **파일 업로드**:
   * **과업지시서 PDF** 영역과 **온보딩 가이드 PDF** 영역에 각 문서를 마우스로 **드래그 앤 드롭**하거나 영역을 클릭하여 파일을 선택합니다.
4. **프로세스 기동**:
   * 하단의 **"온보딩 프로세스 가동"** 버튼을 누르면 업로드 및 파이프라인 처리가 비동기로 진행되며 완료 모달창이 출력됩니다.

## 5. 자동 연동 및 RAG 슬랙 챗봇 활용 시나리오

"온보딩 프로세스 가동" 완료 시 백그라운드에서 다음 연동이 연속적으로 자동 실행됩니다.

1. **Google Drive 연동**: 업로드된 두 PDF 파일이 대표님 계정의 구글 드라이브 폴더에 자동 업로드되어 클라우드 백업이 완료됩니다.
2. **RAG 임베딩 빌드**: PDF의 텍스트가 청킹 가이드(500자 단위)에 맞게 정밀 분할된 후 로컬 벡터 DB(`chroma_db/vector_db.json`)에 임베딩 벡터로 자동 축적(지식 증감)됩니다.
3. **슬랙 채널 자동 개설 및 웰컴 가이드 발송**:
   * 슬랙에 `ot-[프로젝트약어]-[근로자이름]` 채널이 자동 개설됩니다.
   * 입력된 이메일 계정으로 해당 채널 및 워크스페이스 초대장이 발송됩니다.
   * 개설된 채널에 PDF 파일 2개가 자동 업로드됩니다.
   * 신입 근로자를 격하게 축하하는 AI 웰컴 인사말이 채널에 올라옵니다.
4. **챗봇 활용**:
   * 신입 근로자는 채널 내에서 `@Auto_Bot1 [과업에 관한 질문]`을 던질 수 있습니다.
   * 챗봇은 대표님이 방금 대시보드에 업로드한 PDF 자료를 기반으로 지식을 검색(RAG)하여 실시간으로 정확하게 답변을 제공합니다.

## 6. 문제 해결 및 팁 (Troubleshooting)

### 6.1 구글 드라이브 폴더 ID 에러 발생 시
- `.env`에 입력하는 `GOOGLE_DRIVE_FOLDER_ID` 값에 전체 주소(예: `.../folders/1vgWkV5...`) 또는 쿼리 스트링(`?usp=sharing`)이 섞여 들어가더라도, 시스템이 내부적으로 **자동 정제하여 순수 폴더 ID만 추출**하므로 안심하고 사용하셔도 됩니다.

### 6.2 신규 슬랙 채널이 대표님 화면에 보이지 않는 문제
- 봇 토큰(`xoxb-`) 권한으로 공개 채널을 신설하면 최초 생성 시 봇 본인만 채널에 들어갑니다.
- 이를 해결하기 위해 본 시스템은 **`.env`의 `SMTP_USER`에 등록된 대표님(혹은 관리자)의 이메일 계정을 조회하여 채널 개설 즉시 채널로 동시 자동 초대**하도록 고도화되었습니다. 따라서 채널 생성 즉시 대표님의 슬랙 사이드바에도 채널이 정상 노출됩니다.

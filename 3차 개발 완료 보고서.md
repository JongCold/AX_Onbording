# 3차 개발 완료 보고서: 근로자 슬랙 채널 합류 시점 온보딩 자동화 및 RAG 지식 선택적 삭제 기능 개선

본 보고서는 신입 근로자의 슬랙(Slack) 초대 수락 및 채널 합류(실제 입장) 시점의 온보딩 자동화 프로세스 개선과, RAG 시스템에 축적된 지식 파일을 선택적으로 관리하고 삭제할 수 있는 기능 추가를 위한 3차 개발 완료 내역을 정리한 문서입니다.

---

## 1. 개발 배경 및 목적
1. **슬랙 합류 시점 온보딩 지연 발송**:
   - **기존 문제점**: 웹 대시보드에서 신입 근로자를 등록하는 시점에, 근로자가 슬랙 초대장을 수락하기도 전에 프로젝트 채널에 과업지시서와 온보딩 가이드 PDF 파일이 업로드되고 환영 메시지가 발송되었습니다. 이로 인해 근로자가 채널에 입장했을 때 이전 메시지나 파일을 인지하기 어려운 프로세스상의 한계가 존재했습니다.
   - **개선 목표**: 근로자가 가입 이메일의 초대를 수락하고 실제 슬랙 워크스페이스에 참여하여 프로젝트 채널에 가입(합류)한 순간("Auto_Bot1님에 의해 추가되었습니다" 문구 이후)에 정확한 개별 버전의 파일(`.verxx.pdf`)들을 채널에 공유하고 RAG 기반 환영 메시지를 발송하도록 프로세스를 변경합니다.
   
2. **RAG 지식 베이스 선택적 관리 및 청소**:
   - **기존 문제점**: 과업지시서가 변경되거나 온보딩 내용이 수정되어 새로운 버전의 PDF를 업로드할 경우, RAG 시스템에 구버전과 신버전의 지식이 공존하여 챗봇 질의 시 모호하거나 오래된 답변이 발생할 우려가 있었습니다.
   - **개선 목표**: 이전에 인덱싱되었던 개별 PDF 파일 목록을 한눈에 조회하고, 필요 시 특정 문서를 RAG 벡터 데이터베이스(`vector_db.json`)에서 영구히 삭제(청소)할 수 있는 관리 UI와 백엔드 API를 추가하여 지식 정보의 무결성을 유지합니다.

## 2. 주요 수정 내용

### 2.1 슬랙 합류 시점 온보딩 지연 구현
- **대기 사원 정보 스키마 확장 (`main.py`)**:
  - `add_pending` 함수를 수정하여 미가입 근로자의 정보(`name`, `channel_name`) 외에 `task_pdf_path`, `task_filename`, `onboard_pdf_path`, `onboard_filename` 정보를 추가적으로 JSON 파일(`pending_onboardings.json`)에 저장할 수 있도록 변경했습니다.
- **온보딩 요청 백그라운드 파이프라인 분기 처리 (`main.py`)**:
  - 초대 대상 근로자가 슬랙 워크스페이스 미가입 상태(`invited=False`)인 경우, 채널에 즉시 PDF를 업로드하고 웰컴 메시지를 발송하는 로직을 건너뜁니다.
  - 대신 가입 대기자 정보 파일에 업로드할 개별 버전의 파일 정보(경로 및 이름)를 포함하여 `add_pending` 함수로 이관한 후, 이메일 가입 초대장을 발송하고 처리를 조기 종료(`return`)하도록 변경했습니다.
- **채널 합류 시점의 파일 업로드 연동 (`main.py`)**:
  - `run_post_join_onboarding` 비동기 함수가 `task_pdf_path`, `task_filename`, `onboard_pdf_path`, `onboard_filename`을 파라미터로 수신할 수 있도록 시그니처를 수정했습니다.
  - 근로자가 신규 가입하여 슬랙 `team_join` 이벤트 웹훅이 수신되면, `pop_pending` 함수를 호출하여 대기자 정보를 인출합니다.
  - 인출된 정보에서 업로드 대상 파일의 고유 경로들을 파싱하여 `run_post_join_onboarding` 태스크의 인자로 전달합니다.
  - `run_post_join_onboarding` 내에서 근로자를 채널에 성공적으로 추가한 직후, 전달받은 정확한 고유 버전의 파일들(`.verxx.pdf`)을 해당 채널에 자동 업로드하고 AI 환영 메시지를 발송합니다.

### 2.2 RAG 지식 베이스 선택적 삭제 기능 구현
- **RAG 엔진 핵심 메서드 추가 (`rag_service.py`)**:
  - `list_indexed_sources()`: `self.documents_db` 내 청크들의 `metadata.source` 값을 고유하게 집계하고 정렬하여 반환하는 메서드입니다.
  - `delete_source(source_name)`: `source_name`과 일치하는 모든 임베딩 청크를 `self.documents_db`에서 제외(필터링)하고, `vector_db.json`을 영구 갱신 및 저장하는 메서드입니다.
- **백엔드 API 엔드포인트 구현 (`main.py`)**:
  - `GET /rag/sources`: 현재 RAG DB에 등록된 고유 지식소스 파일 목록을 반환합니다.
  - `DELETE /rag/sources?source=파일명`: 특정 지식 파일명을 수신하여 `rag_service.delete_source`를 실행하고 결과를 반환합니다.
- **대시보드 프론트엔드 UI/UX 반영 (`static/` 폴더 내)**:
  - **`index.html`**: 메인 폼 하단에 RAG 지식 베이스 관리용 카드 디자인 영역 및 소스 리스트(`ul#sourceList`) 배치.
  - **`style.css`**: 글래스모피즘(Glassmorphism) 스타일을 확장 적용한 RAG 관리 카드, 아이템 리스트 호버 트랜지션, 강렬한 적색 삭제 버튼 스타일 추가.
  - **`app.js`**: 대시보드 진입 시 RAG 지식소스 자동 로드, 삭제 버튼 클릭 시 Confirm 경고 창을 통해 삭제 처리를 수행하는 비동기 fetch 호출 바인딩. 온보딩 폼 가동 성공 시 RAG 관리 리스트도 자동으로 최신 갱신되도록 연계.

## 3. 기능 검증 시나리오

1. **대기 대상 근로자(미가입 이메일) 등록 테스트**:
   - 웹 UI에서 신규 근로자를 등록했을 때, 슬랙 채널이 생성되고 이메일 초대장이 발송되나, 생성된 채널에는 PDF 파일과 환영 메시지가 미리 올라가지 않는지 확인.
   - `pending_onboardings.json` 파일에 등록 시 첨부했던 고유 버전의 PDF 파일 정보들이 정확히 임시 저장되는지 검증.

2. **근로자 가입 및 합류 검증**:
   - 수신된 이메일 초대 링크를 통해 가입을 완료하고 채널에 입장(합류) 시점에 "Auto_Bot1님에 의해 추가되었습니다" 알림이 발생한 직후, 대기 정보에 저장되어 있던 고유 버전(`.verxx.pdf`)의 과업지시서와 온보딩 가이드가 자동으로 채널에 업로드되는지 확인.
   - 업로드 완료와 동시에 `@Auto_Bot1`을 활용한 실시간 업무 질문 방법이 포함된 AI 환영 메시지가 채널에 출력되는지 최종 확인.

3. **RAG 지식 선택적 삭제 검증**:
   - 웹 대시보드 화면 하단 "RAG 지식 베이스 관리" 목록에 현재 RAG에 등록된 파일명이 리스팅되는지 확인.
   - 특정 지식 파일 옆의 '삭제' 버튼을 클릭하여 DB 청소 수행.
   - 서버 터미널 로그(`🗑️ [RAG DB Cleanup] ... 삭제 완료`) 및 로컬 `vector_db.json` 파일에서 해당 소스 매핑 데이터가 삭제되었는지 검증.
   - 해당 파일과 관련된 내용을 슬랙에서 질문했을 때, 챗봇이 답변을 찾지 못한다고 대응하는지 확인하여 지식 삭제 효과 검증.

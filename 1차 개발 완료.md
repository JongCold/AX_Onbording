# 1차 개발 완료 보고서 및 코드 히스토리

본 문서는 신규 단기 근로자 등록 웹 대시보드 구축 및 RAG(검색 증강 생성)/구글 드라이브/SMTP 초대장 이메일 발송 자동화 파이프라인 구축 완료를 정리한 1차 개발 보고서입니다.

---

## 1. 1차 개발 완료 아키텍처 및 프로세스 흐름

고도화된 단기 근로자 온보딩 자동화 프로세스는 아래 단계를 거쳐 완전 자동으로 작동합니다.

1. **대상 슬랙 채널 동적 연동**:
   - 관리자(대표님)가 웹 브라우저(`http://localhost:8000/`)에 접속하면, 시스템이 슬랙 워크스페이스의 활성화된 **공개 채널 목록을 실시간 조회**하여 "대상 슬랙 채널 선택" 드롭다운 메뉴로 뿌려줍니다.
2. **근로자 등록 및 PDF 업로드**:
   - 대표자가 근로자의 이름, 이메일 주소를 입력하고, 해당 근로자가 배정될 슬랙 채널을 드롭다운에서 선택합니다.
   - **과업지시서 PDF** 및 **온보딩 가이드 PDF**를 드래그 앤 드롭으로 업로드한 뒤 제출합니다.
3. **백그라운드 자동 연동 파이프라인 가동**:
   - **Google Drive**: 업로드된 두 PDF 문서가 지정된 대표님 구글 드라이브 폴더에 자동 저장 및 클라우드 백업됩니다. (ID 내 쿼리 파라미터 자동 정제 탑재)
   - **RAG 임베딩 빌드**: PDF가 500자 단위 청크로 분할되어 `Ollama(nomic-embed-text)`를 통해 로컬 벡터 DB(`vector_db.json`)에 임베딩 벡터로 자동 반영됩니다. (Ollama 버전별 응답 포맷 통합 지원)
   - **대표자 자동 초대**: 대표자의 이메일(`kjhkjh10114@gmail.com`)이 신규 채널로 자동 초대되어, 대표님 슬랙 화면에서도 개설 채널이 즉시 노출됩니다.
   - **신입사원 가입 링크 발송**: 신입사원이 워크스페이스 비회원인 경우, `SLACK_INVITE_URL`에 담긴 슬랙 가입 링크를 포함하여 **세련된 HTML 양식의 가입 초대 이메일을 SMTP로 자동 발송**합니다.
4. **실시간 가입 동기화 및 즉각 채널 매핑**:
   - 신입사원이 이메일의 초대장 버튼을 눌러 슬랙에 가입(`team_join`)하는 즉시, 봇이 가입자의 고유 `User ID`를 획득하여 **지연 시간 없이 즉시 채널로 초대를 집행**합니다.
   - 초대 직후 채널에 PDF 파일 2개가 자동 배포되며, Gemini API 기반의 환영 온보딩 메시지가 채널에 자동 발송됩니다.
5. **RAG 챗봇 구동**:
   - 채널에 합류한 신입사원이 `@Auto_Bot1 [질문]` 형태로 물어보면, 대표님이 대시보드에 업로드한 최신 지식을 RAG로 탐색해 24시간 실시간 답변을 지원합니다.
